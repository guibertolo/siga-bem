## Acceptance Criteria

1. Arquivo `lib/copilot/tools/ranking-caminhoes-por-lucro.ts` criado:
   - **schema (Zod):** `{ periodo: string, ordem?: 'crescente' | 'decrescente', top_n?: number }`. Default ordem = `crescente` (prejuizo primeiro). Default top_n = 5. Max top_n = MAX_TOOL_ROWS.
   - **execute:** para cada caminhao do(s) tenant(s), calcula `SUM(viagem.valor_frete_centavos)` no periodo menos `SUM(gasto.valor)` no periodo = `lucro_centavos`. Ordena conforme ordem. Retorna `{ caminhoes: [{ id, placa, modelo, receita_centavos, gasto_centavos, lucro_centavos }] }`.
   - Tabelas: `viagem`, `gasto`, `caminhao`. Joins por `caminhao_id`. Filtro de data em `viagem.data_saida` e `gasto.data`.
2. Arquivo `lib/copilot/tools/ranking-viagens-por-margem.ts` criado:
   - **schema (Zod):** `{ periodo: string, ordem?: 'crescente' | 'decrescente', top_n?: number }`.
   - **execute:** para cada viagem do periodo, calcula `margem = (valor_frete_centavos - SUM(gasto.valor where gasto.viagem_id = viagem.id)) / valor_frete_centavos`. Retorna `{ viagens: [{ id, origem, destino, data_saida, valor_frete_centavos, gasto_total_centavos, margem_percentual }] }`.
   - Tabelas: `viagem`, `gasto` (via `gasto.viagem_id`).
3. Ambas respeitam RLS via `createClient()` SSR e aplicam `.in('empresa_id', empresaIds)` nas queries de viagem/gasto.
4. Ambas aplicam `.limit(MAX_TOOL_ROWS)` no resultado final (top_n e cap sobre isso).
5. Valores monetarios em centavos no I/O. Percentuais como `number` (0.0 a 1.0 ou 0 a 100 — decidir e documentar).
6. Testes de integracao em `__tests__/copilot/tools/ranking-caminhoes.test.ts` e `ranking-viagens.test.ts` validando shape e ordenacao.
7. Try/catch em cada `execute`, `logger.error` com contexto, re-throw.
8. `npm run lint && typecheck && test && build` passam.

## Dev Notes

### Agregacao
O caminho mais simples e agregar client-side apos uma query com `select('*, gasto(*)')` ou similar. Se o volume for alto, considerar RPC no Supabase — mas **so criar RPC se realmente necessario** (CON-4: zero mudanca de schema no MVP).

### Viagens canceladas
Excluir viagens com `status = 'cancelada'` do ranking de margem (nao representam receita real).

### Caminhoes sem viagem no periodo
Incluidos caminhoes com so gasto e zero viagens (lucro negativo). Caminhoes que nao tiveram nem viagem nem gasto no periodo sao excluidos (nao movimentaram nada, nao entram no ranking).

### Convencao de percentual T3 (recomendada pelo @po)
`margem_percentual` = integer de 0-100 (rounded), pode ser negativo quando a viagem deu prejuizo. Documentado no topo do arquivo `ranking-viagens-por-margem.ts`. Zero-frete viagens sao filtradas no SQL (`.gt('valor_frete_centavos', 0)`) pra evitar divisao por zero.

### Safety cap no T3
`RANKING_LOAD_CAP = 500` viagens carregadas por call. Tenants comuns do FrotaViva tem ordem de 10-50 viagens/mes — cap e generoso. Se superado, a flag `limite_carregamento_atingido = true` sobe e o LLM pode avisar. Nao rebaixa para erro — o ranking ainda retorna resultados uteis do subset.

### Jest config — ignorar helpers
Adicionei `testPathIgnorePatterns: ['/node_modules/', '/__helpers__/']` ao `jest.config.ts` pra jest nao tentar executar `__helpers__/mock-supabase.ts` como test suite. Impacto zero nos testes existentes.

### Helper compartilhado
`__tests__/lib/copilot/tools/__helpers__/mock-supabase.ts` centraliza o `createChain()` thenable. 9.2 ainda usa o helper inline antigo (nao refatorei pra nao expandir escopo). 9.3+ usam o compartilhado.
