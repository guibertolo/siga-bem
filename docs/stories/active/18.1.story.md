## Story
**As a** dono de frota,
**I want** registrar vencimento do documento (CRLV), status do IPVA, valor pago e comprovante no cadastro de cada caminhao,
**so that** eu tenha controle completo da regularizacao da frota sem precisar de planilha externa.

## Epic
Epic 18 — Documentacao de Veiculos.

## Acceptance Criteria

1. Migration adiciona 4 colunas em `caminhao`:
   - `doc_vencimento` (`date`, nullable) — data de vencimento do CRLV
   - `ipva_pago` (`boolean`, default `false`) — se o IPVA do ano corrente foi quitado
   - `ipva_valor_centavos` (`integer`, nullable) — valor pago em centavos
   - `ipva_comprovante_url` (`text`, nullable) — URL do comprovante no Storage
2. Formulario de edicao do caminhao exibe os 4 campos novos em secao "Documentacao".
3. Campo `doc_vencimento` usa date picker com mascara DD/MM/AAAA.
4. Campo `ipva_valor_centavos` usa mascara R$ (mesmo padrao dos gastos).
5. Upload do comprovante usa Supabase Storage (bucket `comprovantes`, path `ipva/{caminhao_id}/`).
6. RLS: mesmas regras do caminhao (dono/gestor da empresa).
7. Listagem de caminhoes mostra badge de alerta quando `doc_vencimento` < hoje + 30 dias.
8. Testes unitarios para migration e validacoes.
9. `npm run lint && typecheck && test` passam.
