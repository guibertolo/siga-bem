## Story
**As a** dono,
**I want** ver a media de preco de combustivel por UF calculada a partir dos lancamentos reais dos motoristas,
**so that** eu possa identificar as regioes mais baratas e caras para abastecer e orientar os motoristas sobre as melhores opcoes de rota para reducao de custo.

## Acceptance Criteria

1. Existe uma pagina ou secao acessivel em `/dashboard/configuracoes/combustivel` (ou equivalente existente da Story 3.3) que exibe a media de preco por UF.
2. Acesso restrito a `role = dono`. Tentativa de acesso com `role = motorista` resulta em redirect com mensagem de acesso negado. (Correcao @po: admin NAO acessa, apenas dono.)
3. A exibicao usa dados da view `vw_media_combustivel_regiao` criada na Story 5.1, agrupada por `uf_abastecimento` e `tipo_combustivel`.
4. Para cada UF com dados, exibe: UF/Estado, tipo de combustivel, preco medio (R$/L), preco minimo, preco maximo, quantidade de abastecimentos, data do abastecimento mais recente.
5. O preco medio e DERIVADO da view (`(valor::numeric / 100) / litros` agregado) — nao e campo armazenado.
6. Quando uma UF possui menos de 3 abastecimentos, exibe indicador visual de "poucos dados" (badge amarelo ou asterisco) para sinalizar que a media pode nao ser representativa.
7. A pagina exibe estado vazio quando nao ha abastecimentos com `uf_abastecimento` preenchido, com mensagem orientativa.
8. Dados atualizados sem cache manual: cada acesso busca dados atuais da view.
9. A view `vw_media_combustivel_regiao` e usada diretamente (sem RPC adicional).
10. Layout responsivo em mobile (375px), dados em cards (nao tabela horizontal com scroll), texto minimo 14px.

## Dev Notes

### Acesso: SOMENTE role=dono (correcao @po)
Diferente da versao anterior que permitia admin, agora SOMENTE `role = dono`:
```typescript
const { data: profile } = await supabase
  .from('usuario')
  .select('role')
  .single()

if (profile?.role !== 'dono') {
  return { error: 'Acesso restrito ao dono da empresa' }
}
```

### View disponivel (criada na Story 5.1)
```
vw_media_combustivel_regiao
  empresa_id            UUID
  uf_abastecimento      CHAR(2)
  tipo_combustivel      combustivel_tipo
  total_abastecimentos  BIGINT
  preco_medio_litro     NUMERIC    -- derivado: (valor/100)/litros agregado
  preco_min_litro       NUMERIC
  preco_max_litro       NUMERIC
  total_litros          NUMERIC
  total_valor_centavos  BIGINT
  primeira_data         DATE
  ultima_data           DATE
```

### Query da server action
```typescript
const { data, error } = await supabase
  .from('vw_media_combustivel_regiao')
  .select('*')
  .order('uf_abastecimento', { ascending: true })
```
A RLS da view herda de `gasto` e filtra automaticamente por `empresa_id`.
