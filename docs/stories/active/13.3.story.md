## Story
**Como** dono de cegonha usuario do FrotaViva,
**Quero** que quando eu clicar em "acertar todas pendentes" de um motorista, o periodo do fechamento ja venha pre-preenchido automaticamente da viagem mais antiga pendente ate a mais recente,
**Para que** eu nao perca tempo calculando datas mentalmente e nao esqueca nenhuma viagem no acerto.

## Acceptance Criteria

1. Na tela de acerto em lote (quando o dono seleciona "acertar todas pendentes" de um motorista), o periodo (`data_inicio` e `data_fim`) nao tem mais seletor de data manual. Os campos sao calculados automaticamente.
2. `data_inicio` = data da viagem mais antiga pendente (nao esta em fechamento_item com fechamento pago) do motorista selecionado.
3. `data_fim` = data da viagem mais recente pendente do mesmo motorista.
4. Se o motorista nao tem viagens pendentes, a UI mostra mensagem amigavel: "Esse motorista nao tem viagens pendentes para acerto" e bloqueia o botao de confirmar.
5. A UI mostra o periodo calculado em formato legivel: "Periodo: DD/MM/AAAA ate DD/MM/AAAA (N viagens)".
6. A UI lista todas as viagens que vao entrar no fechamento (titulo/rota + data + valor) para o dono validar visualmente antes de confirmar.
7. Ao confirmar, o fechamento e criado com o periodo calculado e todas as viagens pendentes sao lancadas como `fechamento_item` tipo `viagem`.
8. Se o dono quer mudar o periodo manualmente (caso edge), ha um botao "Ajustar manualmente" que mostra os campos de data (fallback para fluxo antigo). Este botao e secundario, nao destaque.
9. `npm run typecheck && npm run lint && npm run build` passam.
10. Testes de integracao cobrem: motorista com 0 pendentes / com 1 pendente / com N pendentes / com pendentes em anos diferentes.
