## Story
**As a** usuario do FrotaViva que esqueceu sua senha,
**I want** solicitar a redefinicao de senha via email e definir uma nova senha atraves de um link seguro,
**so that** eu possa recuperar acesso a minha conta sem precisar acionar o dono da empresa.

## Acceptance Criteria

1. Na pagina de login (`app/(auth)/login/page.tsx`), existe um link "Esqueci minha senha" abaixo do formulario de login por senha.
2. O link "Esqueci minha senha" e visivel apenas quando o modo de login ativo e "password" (senha), nao quando estiver em modo "magic-link".
3. O link redireciona para `/auth/recuperar-senha` (ou exibe um formulario inline na propria tela de login -- ver Dev Notes).
4. Na pagina/formulario de recuperacao, existe um campo "Email" e um botao "Enviar link de recuperacao".
5. Ao submeter o formulario, a server action chama `supabase.auth.resetPasswordForEmail(email, { redirectTo: '{URL}/auth/reset-password' })`.
6. Independentemente de o email existir ou nao no sistema, exibe a mensagem de sucesso: "Se este email estiver cadastrado, voce recebera um link de recuperacao em breve." (previne enumeracao de usuarios).
7. Existe a pagina `/auth/reset-password` (rota em `app/(auth)/reset-password/page.tsx`) para definicao da nova senha.
8. A pagina `/auth/reset-password` exibe dois campos: "Nova senha" e "Confirmar nova senha".
9. A pagina `/auth/reset-password` valida o token de reset presente na URL (parametro de hash do Supabase). Se o token for invalido ou expirado, exibe: "Link de recuperacao invalido ou expirado. Solicite um novo."
10. Ao submeter a nova senha, chama `supabase.auth.updateUser({ password: novaSenha })` com o cliente de sessao (o token de reset ja estabelece a sessao temporaria do Supabase).
11. A nova senha deve ter minimo de 8 caracteres. Mensagem: "A senha deve ter pelo menos 8 caracteres".
12. "Nova senha" e "Confirmar nova senha" devem ser identicas. Mensagem: "As senhas nao coincidem".
13. Apos redefinir com sucesso, exibe mensagem: "Senha redefinida com sucesso!" e redireciona para `/login` apos 3 segundos.
14. A pagina `/auth/reset-password` funciona sem autenticacao previa (usuario esta fora do dashboard).

## Dev Notes

### Abordagem: Inline vs Nova Rota
[AUTO-DECISION] Implementar como secao inline na pagina de login (mostrar/ocultar com estado React), sem nova rota `/auth/recuperar-senha`. Razao: simplifica o fluxo de navegacao, o usuario ja esta na tela de login e nao precisa de redirect extra para uma funcionalidade simples. A pagina de login ja e um Client Component.

Implementacao sugerida:
```typescript
const [modo, setModo] = useState<'login' | 'recuperar'>('login');
```
E alternar entre o formulario de login e o formulario de recuperacao no mesmo componente.

### Supabase Auth Reset Flow
O Supabase envia um email com um link no formato:
`{redirectTo}#access_token=...&type=recovery`

O cliente Supabase detecta automaticamente o token de recovery na URL ao inicializar. A pagina `/auth/reset-password` deve chamar:
```typescript
const supabase = createClientBrowserClient();
// O token ja foi processado pelo Supabase client automaticamente
// Verificar se ha sessao de recovery valida:
const { data: { session } } = await supabase.auth.getSession();
if (!session || session.user.aud !== 'authenticated') {
  // token invalido ou expirado
}
```

### redirectTo em Producao
A URL de redirect deve ser absoluta. Usar variavel de ambiente `NEXT_PUBLIC_APP_URL` se existir, ou `window.location.origin` no client. Para server action, usar o header `origin` da requisicao.

### Pagina /auth/reset-password
Esta pagina nao faz parte do grupo `(dashboard)` -- deve estar em `(auth)`. Verificar se o grupo `(auth)` tem layout proprio. O layout de auth nao exige autenticacao.

### Existencia do Modo Magic Link
A pagina de login ja tem modo `magic-link` (conforme leitura do arquivo). O link "Esqueci minha senha" deve ser condicional ao modo `password` (AC 2).
