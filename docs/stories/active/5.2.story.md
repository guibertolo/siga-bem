## Story
**As a** motorista,
**I want** lancar uma nota de abastecimento de combustivel diretamente na tela da minha viagem ativa,
**so that** eu consiga registrar litros, valor, posto, cidade e foto do comprovante sem sair do contexto da viagem, com o caminhao e viagem ja preenchidos automaticamente.

## Acceptance Criteria

1. Na pagina de detalhe de viagem (`/dashboard/viagens/[id]`) com `status = 'em_andamento'`, existe um botao "Registrar Abastecimento" visivel e acessivel (minimo 44x44px, contraste adequado).
2. Ao clicar no botao, abre um formulario (modal ou pagina dedicada) com os seguintes campos pre-preenchidos e bloqueados (readonly): viagem (origem/destino), caminhao (placa), motorista (nome do usuario logado), categoria = "Combustivel", data = hoje.
3. O formulario exibe os seguintes campos editaveis obrigatorios: Litros (decimal, ex: 45.70), Preco por Litro em R$/L (decimal, ex: 6.45), Cidade do abastecimento (texto), Estado/UF (select com 27 UFs brasileiras).
4. O formulario exibe os seguintes campos editaveis opcionais: Nome do Posto (texto livre, max 200 chars), Bandeira (texto livre, ex: Shell, BR), Leitura do Odometro em km (inteiro >= km_saida da viagem), Observacao (textarea, max 500 chars).
5. O campo "Valor Total (R$)" e calculado automaticamente em tempo real: `valor_total = litros * preco_litro`, exibido como readonly. Motorista NAO digita o valor total, o sistema calcula.
6. O campo Tipo de Combustivel e um select com opcoes "Diesel S10" e "Diesel Comum", com default "Diesel S10".
7. Ao submeter, o sistema realiza duas operacoes em transacao: (a) INSERT em `gasto` com `categoria_id` = Combustivel, `valor` = valor_total em centavos, `viagem_id`, `motorista_id`, `caminhao_id`, `empresa_id`, `km_registro` = odometro; (b) INSERT em `abastecimento_detalhe` com `gasto_id` = novo ID, `litros`, `preco_litro`, `tipo_combustivel`, `km_odometro`, `posto`, `bandeira`, `cidade`, `estado`.
8. Apos salvar com sucesso, o motorista e redirecionado de volta para a pagina da viagem e ve uma mensagem de sucesso (toast/alert).
9. Validacoes de formulario: litros > 0 e <= 999.99, preco_litro > 0, cidade nao vazia, estado valido (2 letras maiusculas), odometro (se preenchido) >= km_saida da viagem.
10. Em caso de erro na API/banco, exibe mensagem de erro descritiva sem perder os dados preenchidos (sem reset do form).
11. Layout mobile-first: coluna unica, sem scroll horizontal, inputs com `inputMode="decimal"` para litros e preco, `inputMode="numeric"` para odometro, todos com altura minima de 48px.
12. O botao "Registrar Abastecimento" nao aparece para viagens com `status != 'em_andamento'`.
13. Upload de foto do comprovante esta disponivel no mesmo formulario, reutilizando o componente `ComprovantesUpload.tsx` existente (Story 2.2). A foto e vinculada ao `gasto_id` criado na transacao.

### Fluxo de dados da transacao
```typescript
// Server action createAbastecimento
const { data: gasto } = await supabase
  .from('gasto')
  .insert({
    empresa_id,          // do usuario autenticado
    categoria_id,        // UUID da categoria 'Combustivel' (buscar por nome)
    motorista_id,        // do usuario autenticado
    caminhao_id,         // da viagem
    viagem_id,           // da viagem
    valor: Math.round(litros * preco_litro * 100),  // centavos
    data: new Date().toISOString().split('T')[0],
    km_registro: km_odometro ?? null,
  })
  .select('id')
  .single()

await supabase
  .from('abastecimento_detalhe')
  .insert({
    gasto_id: gasto.id,
    litros,
    preco_litro,
    tipo_combustivel,
    km_odometro: km_odometro ?? null,
    posto: posto ?? null,
    bandeira: bandeira ?? null,
    cidade,
    estado,
    observacao: observacao ?? null,
  })
```

### Calculo de valor_total (client-side, em tempo real)
```typescript
// valor_total em R$ = litros * preco_litro
// Exibir como currency formatada, readonly
const valorTotal = (litros && precoLitro) ? (litros * precoLitro).toFixed(2) : '0.00'
```
O valor armazenado em `gasto.valor` e `Math.round(litros * precoLitro * 100)` (centavos inteiros, CON-003).

### UX — Mobile-First (publico 60+)
- Todos os inputs numericos com `inputMode="decimal"` (litros, preco_litro) ou `inputMode="numeric"` (odometro)
- Altura minima de 48px para todos os inputs e botoes
- Botao de acao principal com pelo menos 56px de altura
- Label claramente associado a cada input (htmlFor)
- Nao usar placeholders como substitutos de labels
- Texto de erro abaixo do campo, nao popup flutuante

### Buscar UUID da categoria Combustivel
NAO usar UUID hardcoded. Fazer:
```typescript
const { data: cat } = await supabase
  .from('categoria_gasto')
  .select('id')
  .eq('nome', 'Combustivel')
  .single()
```
Cachear no componente ou server action.

### Upload de foto
Reutilizar `ComprovantesUpload.tsx` que recebe `gastoId` como prop. O upload deve ocorrer APOS o INSERT do gasto (pois precisa do `gasto_id`). Fluxo: (1) salvar gasto + detalhe → (2) se foto, fazer upload via `ComprovantesUpload` passando o `gastoId` retornado.
