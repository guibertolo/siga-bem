## Story
**As a** motorista,
**I want** registrar um abastecimento de combustivel diretamente na tela da minha viagem ativa, com upload de foto do comprovante no mesmo formulario,
**so that** eu consiga registrar litros, valor total, posto, UF e foto sem sair do contexto da viagem, com caminhao e viagem ja preenchidos automaticamente, e o preco por litro exibido sem eu precisar calcular.

## Acceptance Criteria

1. Na pagina de detalhe de viagem (`/dashboard/viagens/[id]`) com `status = 'em_andamento'`, existe um botao "+ Registrar Abastecimento" visivel, com minimo 48x48px e contraste adequado (WCAG AA).
2. O botao nao aparece para viagens com `status != 'em_andamento'`.
3. Ao clicar no botao, abre um formulario (modal ou pagina dedicada) com campos pre-preenchidos e bloqueados (readonly): viagem (origem/destino), caminhao (placa), motorista (nome do usuario logado), categoria = "Combustivel", data = hoje.
4. O formulario exibe campos editaveis obrigatorios: Litros (decimal NUMERIC 10,3, ex: 245.750), Valor Total em R$ (decimal, ex: 1.584,00), UF do abastecimento (select com 27 UFs brasileiras).
5. O formulario exibe campos editaveis opcionais: Posto/Local (texto livre, max 200 chars), Tipo de Combustivel (select "Diesel S10" / "Diesel Comum", default "Diesel S10"), Leitura do Odometro em km (inteiro >= km_saida da viagem quando preenchida), Observacao (textarea, max 500 chars).
6. O campo "Preco por Litro (R$/L)" e calculado automaticamente em tempo real: `preco_litro = valor_total / litros`, exibido como readonly com 3 casas decimais. Motorista NAO digita preco_litro — o sistema deriva.
7. Upload de foto do comprovante disponivel no mesmo formulario, reutilizando o componente `ComprovantesUpload.tsx` existente (Stories 2.x). Foto e vinculada ao `gasto.id` criado apos o INSERT.
8. Ao submeter, o sistema realiza um INSERT em `gasto` com os campos: `empresa_id`, `categoria_id` (UUID de "Combustivel" buscado por nome), `motorista_id`, `caminhao_id`, `viagem_id`, `valor` (centavos = round(valor_total * 100)), `data`, `km_registro`, `litros`, `tipo_combustivel`, `posto_local`, `uf_abastecimento`.
9. Apos salvar com sucesso, se houver foto, o upload e disparado usando o `gasto_id` retornado. Em seguida, o motorista e redirecionado para a pagina da viagem com toast de sucesso.
10. Validacoes de formulario (client-side e server-side): litros > 0 e <= 9999.999, valor_total > 0, uf_abastecimento valida (2 letras maiusculas, valor do select), odometro (se preenchido) deve ser inteiro positivo.
11. Em caso de erro na API/banco, exibe mensagem de erro descritiva sem resetar o formulario (dados digitados preservados).
12. Layout mobile-first: coluna unica, sem scroll horizontal, inputs numericos com `inputMode="decimal"` (litros, valor), `inputMode="numeric"` (odometro), altura minima de 48px em todos inputs e botoes, label claramente associado a cada input (htmlFor), erros abaixo do campo (nao popup flutuante).
13. Categoria "Combustivel" pré-selecionada e bloqueada — motorista nao pode mudar.

## Dev Notes

### Abordagem: INSERT unico em gasto
Diferente da versao anterior (tabela separada com transacao dupla), agora e um INSERT unico em `gasto` com os 4 campos novos:
```typescript
// Server action createAbastecimento
const { data: gasto, error } = await supabase
  .from('gasto')
  .insert({
    empresa_id,          // do usuario autenticado
    categoria_id,        // UUID de 'Combustivel' (buscar por nome)
    motorista_id,        // do usuario autenticado
    caminhao_id,         // da viagem
    viagem_id,           // parametro
    valor: Math.round(valorTotal * 100),  // centavos
    data: new Date().toISOString().split('T')[0],
    km_registro: kmOdometro ?? null,
    litros,                          // NOVO
    tipo_combustivel,                // NOVO
    posto_local: postoLocal ?? null, // NOVO
    uf_abastecimento: uf,            // NOVO
  })
  .select('id')
  .single()
```

### Preco por litro — calculo client-side (tempo real)
```typescript
// Campo readonly, atualizado a cada keystroke em litros ou valor_total
const precoLitro = litros > 0 && valorTotal > 0
  ? (valorTotal / litros).toFixed(3)
  : null
// Exibir: "R$ 6,449/L" (formato pt-BR)
```

NAO usar UUID hardcoded. Cachear na server action:

const { data: cat } = await supabase

3. Redirecionar para pagina da viagem com toast
