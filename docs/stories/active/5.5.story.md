## Story
**As a** dono,
**I want** acessar um dashboard de BI de combustivel com metricas agregadas por caminhao, motorista e periodo,
**so that** eu possa analisar o custo real de combustivel da frota, identificar o consumo medio (km/l) de cada caminhao e estimar o custo de combustivel de proximas viagens com base no historico real.

## Acceptance Criteria

1. Existe uma rota `/dashboard/bi/combustivel` acessivel apenas para `role = dono` ou `role = admin`. Tentativa de acesso com `role = motorista` resulta em redirect para `/dashboard` com mensagem de acesso negado.
2. A rota `/dashboard/bi` NAO aparece no menu lateral (sidebar) para usuarios com `role = motorista`.
3. A pagina possui filtros no topo: periodo (date range, default: ultimos 30 dias), caminhao (multi-select, todos por default), motorista (multi-select, todos por default), tipo de combustivel (select: Todos / Diesel S10 / Diesel Comum).
4. Ao alterar qualquer filtro, os dados da pagina sao atualizados sem reload completo (client-side filtering ou refetch).
5. A pagina exibe cards de KPIs com: (a) Gasto Total de Combustivel no periodo (R$), (b) Total de Litros abastecidos, (c) Preco Medio por Litro (R$/L), (d) Numero de Abastecimentos.
6. A pagina exibe uma tabela ou lista de "Custo por Caminhao" com dados da view `vw_custo_combustivel_caminhao`: placa, modelo, total abastecimentos, total litros, total gasto (R$), km/l estimado (quando disponivel via odometro).
7. A pagina exibe uma tabela ou lista de "Custo por Motorista" com dados da view `vw_custo_combustivel_motorista`: nome, total abastecimentos, total litros, total gasto (R$), preco medio (R$/L).
8. A pagina exibe um simulador de previsao de custo: o dono digita km estimado e seleciona caminhao, e o sistema exibe o custo estimado de combustivel usando a funcao `fn_estimativa_custo_viagem`. O resultado exibe: litros estimados, custo estimado (R$), consumo usado (km/l) e fonte do dado (historico ou padrao).
9. Quando ha dados insuficientes para um caminhao (sem abastecimentos com km_odometro preenchido), o campo km/l exibe "—" ou "Sem dados de odometro".
10. Quando nao ha abastecimentos no periodo selecionado, todos os cards mostram zero e as tabelas mostram estado vazio com mensagem orientativa.
11. Os dados sao buscados via server actions que chamam as views/funcoes criadas na Story 5.1 (`vw_custo_combustivel_caminhao`, `vw_custo_combustivel_motorista`, `fn_estimativa_custo_viagem`).
12. A pagina e responsiva: em mobile (375px), os cards empilham verticalmente, as tabelas viram cards em lista, os filtros ficam em coluna unica.
13. Nao ha graficos de barras/linhas nesta story (escopo MVP). Dados apresentados em cards e tabelas apenas. Graficos podem ser adicionados em story futura.

### Funcao de estimativa (criada na Story 5.1)
```sql
fn_estimativa_custo_viagem(
  p_empresa_id UUID,
  p_km_estimado INTEGER,
  p_consumo_medio_km_l NUMERIC DEFAULT NULL,
  p_tipo_combustivel combustivel_tipo DEFAULT 'diesel_s10'
) RETURNS TABLE (
  consumo_usado_km_l NUMERIC,
  preco_medio_litro NUMERIC,
  litros_estimados NUMERIC,
  custo_estimado_centavos INTEGER,
  fonte_consumo TEXT,   -- 'parametro_manual' | 'historico_empresa_90d' | 'padrao_cegonheiro'
  fonte_preco TEXT      -- 'combustivel_preco' | 'media_abastecimentos_30d' | 'padrao_nacional'
)
```

Chamar via Supabase RPC:
```typescript
const { data } = await supabase
  .rpc('fn_estimativa_custo_viagem', {
    p_empresa_id: empresaId,
    p_km_estimado: kmEstimado,
    p_tipo_combustivel: tipoCombustivel,
  })
```

### Filtros e queries com periodo
As views nao tem filtro de data embutido — filtrar na query:
```typescript
const { data } = await supabase
  .from('vw_custo_combustivel_caminhao')
  .select('*')
  .gte('primeiro_abastecimento', periodoInicio)
  .lte('ultimo_abastecimento', periodoFim)
```
Atencao: o filtro por data na view filtra pelo `primeiro_abastecimento` e `ultimo_abastecimento`, que sao derivados. Para filtro preciso por periodo, pode ser necessario criar uma query direta em `gasto JOIN abastecimento_detalhe` com filtro em `gasto.data`. Avaliar a melhor abordagem para os KPIs totais.

### KPIs totais — query direta
Para os cards de KPI com filtro preciso por periodo e caminhao:
```typescript
const { data } = await supabase
  .from('gasto')
  .select(`
    valor,
    abastecimento_detalhe ( litros, preco_litro, tipo_combustivel )
  `)
  .gte('data', periodoInicio)
  .lte('data', periodoFim)
  .not('abastecimento_detalhe', 'is', null)
  // adicionar .in('caminhao_id', caminhoesSelecionados) se filtro aplicado
  // adicionar .in('motorista_id', motoristasSelecionados) se filtro aplicado
```
Calcular KPIs no cliente ou em server action.
