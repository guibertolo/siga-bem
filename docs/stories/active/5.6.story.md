## Story
**As a** dono,
**I want** ver a estimativa de custo e margem de lucro para a proxima viagem baseada no historico de rotas similares, e comparar o desempenho financeiro entre viagens na mesma rota,
**so that** eu possa precificar melhor os fretes, identificar rotas não-rentaveis e otimizar a alocacao de caminhoes e motoristas.

## Acceptance Criteria

1. Existe uma pagina ou secao dentro do BI (`/dashboard/bi` ou sub-rota) acessivel APENAS para `role = dono`. Acesso com qualquer outro role resulta em redirect para `/dashboard`.
2. O dono pode inserir os parametros de uma nova viagem: origem (texto livre), destino (texto livre), km estimado (inteiro obrigatorio), caminhao (select opcional), tipo de combustivel (select, default Diesel S10).
3. Ao confirmar os parametros, o sistema exibe a "Estimativa de Custo" com: litros estimados de combustivel, custo estimado de combustivel (R$), consumo km/l utilizado na estimativa, fonte do consumo ("Historico real" ou "Padrao cegonheiro 2,5 km/l").
4. O campo "Valor do Frete (R$)" esta disponivel para o dono preencher (opcional). Quando preenchido, o sistema calcula e exibe a "Margem de Lucro Estimada": valor_frete - custo_estimado_combustivel, em R$ e percentual.
5. Quando nao ha historico de consumo para o caminhao selecionado (sem abastecimentos com odometro preenchido), a estimativa usa 2,5 km/l como padrao e exibe aviso claro: "Sem historico — usando consumo padrao de cegonheiros (2,5 km/l)".
6. Secao "Historico de Rotas Similares" exibe viagens anteriores que combinam origem e destino semelhantes (busca por texto parcial). Para cada viagem historica exibe: data, caminhao, motorista, km realizado, custo total de gasto (todos tipos), custo de combustivel, e lucro real (quando frete cadastrado na viagem).
7. [AUTO-DECISION] Verificar se campo `valor_frete` ou equivalente existe na tabela `viagem`. Se nao existir, a exibicao de lucro real no historico mostra "Frete nao cadastrado" e a estimativa de margem fica disponivel apenas para entrada manual do valor do frete pelo dono. (Reason: a PR de verificacao e mais segura que assumir o campo existe.)
8. Comparativo entre viagens historicas na mesma rota: exibe variacao de custo (minimo, maximo, media) das viagens anteriores na rota.
9. Toda a pagina e responsiva (375px mobile), inputs com altura minima 48px, resultados em cards verticais.
10. Dados filtrados por `empresa_id` via RLS automatica.

### Calculo de estimativa (sem RPC separada)
Diferente da versao do BI de combustivel que usava `fn_estimativa_custo_viagem`, aqui calculamos diretamente na server action para evitar dependencia de function SQL que pode nao existir:

```typescript
// getEstimativaCustoViagem
let consumoKmL = 2.5 // padrao
let fonteConsumo = 'padrao_cegonheiro'

if (caminhaoId) {
  const { data: caminhaoView } = await supabase
    .from('vw_custo_por_caminhao')
    .select('km_por_litro_estimado')
    .eq('caminhao_id', caminhaoId)
    .single()

  if (caminhaoView?.km_por_litro_estimado) {
    consumoKmL = caminhaoView.km_por_litro_estimado
    fonteConsumo = 'historico_real'
  }
}

const litrosEstimados = kmEstimado / consumoKmL
const custoEstimadoCentavos = Math.round(litrosEstimados * precoMedioLitro * 100)
```

### Preco medio de combustivel para estimativa
```typescript
// Tentar preco medio real dos ultimos 30 dias
const { data: mediaPreco } = await supabase
```
