## Story
**As a** usuario do FrotaViva com multiplas empresas,
**I want** trocar de empresa sem precisar fazer logout, via um seletor no sidebar ou header do dashboard,
**so that** eu possa alternar rapidamente entre os CNPJs que gerencio durante a mesma sessao.

## Acceptance Criteria

1. No sidebar, abaixo do logo, exibe o nome da empresa ativa: `nome_fantasia` se disponivel, caso contrario `razao_social`.
2. Usuarios com mais de 1 empresa vinculada veem um dropdown/botao para trocar de empresa. Usuarios com 1 empresa veem apenas o label estatico sem interacao.
3. Ao abrir o seletor, lista todas as empresas ativas vinculadas ao usuario com: razao_social, CNPJ (formatado), role do usuario naquela empresa.
4. A empresa atualmente ativa aparece com indicador visual (check ou destaque) na lista.
5. Ao clicar em outra empresa na lista, o sistema chama `switchEmpresa(empresaId)`, faz `revalidatePath('/')` e recarrega a pagina — todos os dados do dashboard passam a refletir a nova empresa.
6. O servidor valida que o usuario tem vinculo ativo com a empresa destino antes de executar a troca — rejeita silenciosamente com toast de erro se invalido.
7. Durante a troca, o seletor exibe estado de loading (disabled) ate a pagina recarregar.
8. O componente `EmpresaSwitcher` e inserido no layout do dashboard (`app/(dashboard)/layout.tsx`) de forma que aparece em todas as paginas do dashboard.
9. Apos a troca, a URL permanece na mesma pagina (ex: se estava em `/viagens`, continua em `/viagens` apos trocar a empresa).
10. O seletor busca a lista de empresas server-side (no layout) para evitar flash de conteudo no carregamento.

## Dev Notes

### Reutilizacao de switchEmpresa
A server action `switchEmpresa` foi criada na Story 7.2 em `app/(dashboard)/empresa/switch/actions.ts`. Reutilizar sem duplicar.

### Server component no layout
O `layout.tsx` e um Server Component — buscar `getUserEmpresas()` diretamente (sem useEffect ou SWR). Passar como prop para `EmpresaSwitcher` que e Client Component (precisa de interatividade).

### revalidatePath apos troca
`revalidatePath('/')` invalida cache de todas as rotas. O redirect deve manter o `pathname` atual:
```typescript
redirect(pathname) // nao redirecionar para /dashboard
```
Para manter na mesma pagina, a server action pode receber o `pathname` como parametro.

### Design do EmpresaSwitcher
O seletor fica no sidebar, abaixo do logo. Sugestao de layout:
```
[Logo FrotaViva]
[Nome da Empresa v]  <- EmpresaSwitcher (com chevron se multiplas)
[Menu items...]
```
