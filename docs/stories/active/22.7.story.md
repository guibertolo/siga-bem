# Story 22.7: Sentry replay com mascara de dados sensiveis

## Status
Draft

## Executor Assignment
```
executor: "@dev"
quality_gate: "@qa"
```

## Story
**As a** responsavel pela privacidade dos usuarios do FrotaViva,
**I want** que o Sentry Session Replay mascare explicitamente todos os dados sensiveis (CPF, valores financeiros, midias),
**so that** replays de sessao nao exponham informacoes pessoais ou financeiras dos motoristas e donos de frota.

## Epic
Epic 22 — Infra Hardening.

## Origem
Addendum 2026-04-14, item #7

## Acceptance Criteria

1. O arquivo `sentry.client.config.ts` e refatorado para adicionar `Sentry.replayIntegration()` no array `integrations` do `Sentry.init()`. O integration nao existe atualmente no arquivo (verificado: o init atual nao tem o campo `integrations`). A configuracao deve incluir obrigatoriamente:
   - `maskAllInputs: true`
   - `maskAllText: true`
   - `blockAllMedia: true`

2. Com `maskAllInputs: true` ativo, todos os campos `<input>` e `<textarea>` sao automaticamente mascarados pelo Sentry Replay. O atributo `data-sentry-mask` NAO e adicionado em campos de formulario — seria redundante e geraria ruido desnecessario de atributos no DOM.

3. Os seguintes componentes que renderizam valores financeiros ou CPF em elementos nao-input (ex: `<span>`, `<p>`, texto renderizado) recebem `data-sentry-mask` nos elementos container que exibem o dado sensivel:
   - `components/gastos/DespesaViagemSection.tsx` — verificar se campo de valor (`maskCurrency`) e renderizado em elemento de texto ou apenas em input; adicionar `data-sentry-mask` somente se houver elemento de texto com valor real exposto fora de `<input>`.
   - Formularios `GastoForm`, `MotoristaForm`, `FechamentoForm` — idem: adicionar `data-sentry-mask` apenas em elementos nao-input que exibam CPF ou valor monetario real (ex: preview de valor, exibicao de CPF formatado).

4. As opcoes `replaysSessionSampleRate: 0` e `replaysOnErrorSampleRate: 1.0` ja presentes em `sentry.client.config.ts` sao mantidas sem alteracao.

5. `npm run lint && npm run typecheck && npm run build` passam sem erros.

## Technical Notes

- Estado atual de `sentry.client.config.ts` (verificado via leitura do arquivo):
  ```ts
  Sentry.init({
    dsn: process.env.NEXT_PUBLIC_SENTRY_DSN,
    tracesSampleRate: 0.1,
    replaysSessionSampleRate: 0,
    replaysOnErrorSampleRate: 1.0,
    debug: false,
    enabled: process.env.NODE_ENV === 'production',
  });
  ```
  Nao ha `integrations` nem `replayIntegration` — o integration precisa ser criado do zero nesta story.

- `maskAllInputs: true` e o modo mais seguro (mascara tudo por default). Com ele ativo, `data-sentry-mask` em campos de input e redundante — aplicar apenas em elementos de texto que exibam dados sensiveis fora de inputs.

- Para adicionar o integration:
  ```ts
  import * as Sentry from '@sentry/nextjs';
  Sentry.init({
    // ... existente ...
    integrations: [
      Sentry.replayIntegration({
        maskAllInputs: true,
        maskAllText: true,
        blockAllMedia: true,
      }),
    ],
  });
  ```

- `DespesaViagemSection` (`components/gastos/DespesaViagemSection.tsx`): componente de secao de gastos na pagina de detalhe de viagem. Usa `maskCurrency` — verificar se o valor e exibido apenas em `<input>` (ja coberto por `maskAllInputs`) ou tambem em `<span>`/`<p>` de preview.

- Complexidade: S / Risco S (sem alteracao de logica de negocio, apenas configuracao de SDK e atributos HTML).

## Testing Requirements

- Code review: verificar que elementos nao-input com dados sensiveis identificados no AC 3 receberam `data-sentry-mask`.
