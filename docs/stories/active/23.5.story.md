## Story
**As a** dono de frota,
**I want** um relatorio completo das viagens de um motorista em um periodo, com totais, pagamento calculado e ranking na frota,
**so that** eu possa fazer o acerto semanal com o motorista usando numeros confiaveis, sem planilha.

## Epic
Epic 23 (REL) — Relatorios e Upload da Chamada.

## Acceptance Criteria

1. RPC `relatorio_motorista_periodo(p_motorista_id uuid, p_inicio date, p_fim date) RETURNS jsonb` criada em `supabase/migrations/` com `SECURITY INVOKER` (herda RLS do caller).
2. A RPC retorna um objeto JSONB com a estrutura:
   - `header`: `{ motorista_nome, motorista_cpf, empresa_nome, periodo_inicio, periodo_fim, total_viagens, total_km_calculado, total_valor_bruto_centavos, total_pagamento_centavos }` — onde `total_km_calculado` e `SUM(km_chegada - km_saida)` excluindo viagens com km_saida ou km_chegada NULL; `total_pagamento_centavos` e `SUM(ROUND(valor_total * percentual_pagamento / 100.0))` por viagem.
   - `viagens`: array de `{ id, data_saida, data_chegada_real, origem, destino, km_calculado, valor_total_centavos, percentual_pagamento, pagamento_centavos, status, comprovantes: [{ url_signed, storage_path }] }` — onde `km_calculado = km_chegada - km_saida` (null se qualquer dos dois for null); comprovantes sao fotos de gastos vinculados a viagem (`foto_comprovante` via `gasto.viagem_id`); `url_signed` gerada na server action TypeScript (nao no SQL).
   - `caminhoes_usados`: array de `{ placa, modelo }` distintos no periodo (via JOIN `viagem.caminhao_id -> caminhao`).
   - `dias_trabalhados`: integer (COUNT de datas distintas em `data_saida` no periodo).
   - `dias_ociosos`: integer (dias corridos no periodo menos `dias_trabalhados`).
   - `ranking_frota`: `{ posicao, total_motoristas, metrica: 'valor_bruto' }` — posicao do motorista vs outros da mesma empresa por SUM(`valor_total`) em viagens `status = 'concluida'` no periodo.
3. Server Action `gerarRelatorioMotorista` em `app/(dashboard)/relatorios/motorista/actions.ts` que: (a) valida que o `motorista_id` pertence a `empresa_id` da sessao (`motorista.empresa_id`); (b) chama a RPC; (c) gera signed URLs de comprovantes via `supabase.storage.from('comprovantes').createSignedUrl(storage_path, 300)` para cada `foto_comprovante.storage_path` das fotos vinculadas; (d) retorna os dados tipados.
4. Tela `app/(dashboard)/relatorios/motorista/[id]/page.tsx` (Server Component) renderiza os dados em cards empilhados, mobile-first:
   - Header fixo com nome do motorista, periodo e totais principais (total de viagens, km calculado, valor bruto, pagamento motorista).
   - Cards de viagem individualmente listando: data, origem/destino, km calculado (ou "-" se null), valor, percentual_pagamento, pagamento calculado, status (badge colorido), miniatura do comprovante se existir.
   - Cards de "Caminhoes usados no periodo" e "Resumo de dias".
   - Card de ranking com posicao na frota.
5. Botoes "Baixar PDF" e "Baixar Excel" exibidos no topo da tela, desabilitados com tooltip "Aguarde..." enquanto dados carregam. Nao exibir botao "Imprimir" separado.
6. `@media print` na tela converte o layout para formato tabular A4 (sem header fixo, sem botoes, com nome da empresa em texto — nao logo).
7. Defesa-em-profundidade: server action rejeita com erro 403 se `motorista_id` nao pertencer a empresa da sessao autenticada.
8. Signed URLs dos comprovantes geradas com TTL de 300 segundos na server action TypeScript via `supabase.storage.from('comprovantes').createSignedUrl(path, 300)`.
9. Testes de integracao com Supabase local validam que motorista de empresa B nao obtem dados da empresa A.

**Regra de pagamento do motorista:**
`viagem.percentual_pagamento` herda de `motorista.percentual_pagamento` no momento da criacao da viagem (ver `app/(dashboard)/viagens/actions.ts`:linha 267). Calcular na RPC: `ROUND(valor_total * percentual_pagamento / 100.0)::INTEGER AS pagamento_centavos`.

**km_total NAO existe em viagem:**
Calcular como `(km_chegada - km_saida)` — sera null quando qualquer um for null. Exibir "-" na UI. Na RPC SQL:
```sql
CASE WHEN km_saida IS NOT NULL AND km_chegada IS NOT NULL
     THEN km_chegada - km_saida
     ELSE NULL
END AS km_calculado
```

**Calculo de ranking:**
```sql
WITH motorista_rank AS (
  SELECT motorista_id, SUM(valor_total) AS total,
    RANK() OVER (ORDER BY SUM(valor_total) DESC) AS posicao
  FROM viagem
  WHERE empresa_id = (SELECT empresa_id FROM motorista WHERE id = p_motorista_id)
    AND data_saida::date BETWEEN p_inicio AND p_fim
    AND status = 'concluida'
  GROUP BY motorista_id
)
SELECT posicao, (SELECT COUNT(*) FROM motorista_rank) AS total_motoristas
FROM motorista_rank WHERE motorista_id = p_motorista_id
```
