## Story
**Como** desenvolvedor do FrotaViva,
**Quero** eliminar dead code e renomear `multi-actions.ts` para um nome que descreva o que ele realmente e (queries de leitura, nao mutacoes),
**Para que** o codigo pare de mentir sobre sua propria natureza e facilite manutencao futura.

## Acceptance Criteria

1. O arquivo `app/(dashboard)/empresa/switch-actions.ts` e deletado. Qualquer import que apontava para ele e removido ou redirecionado para o modulo substituto.
2. Uma busca global por `switch-actions` nao encontra nenhuma ocorrencia no codigo.
3. Todos os arquivos chamados `multi-actions.ts` sao renomeados para `multi-queries.ts` (alternativa: movidos para `lib/queries/{dominio}.ts`). A escolha do caminho exato e do @dev, mas deve ser consistente em todos os dominios.
4. Todos os imports de `multi-actions` sao atualizados para o novo nome.
5. `npm run typecheck` e `npm run lint` passam.
6. `npm run build` passa sem warnings novos.
7. Dev local funciona: subir dashboard, fazer login, abrir viagens, fechamentos e BI. Tudo carrega sem erro de console.
8. Commit separado por domin ou um commit atomico bem descrito (criterio do @dev, mas o PR/commit descreve exatamente quais arquivos mudaram).
