## Story
**As a** motorista ou dono,
**I want** ver a lista de todos os abastecimentos registrados em uma viagem,
**so that** eu possa acompanhar o historico de combustivel da viagem com data, litros, valor, posto e acesso a foto do comprovante, sabendo que motorista ve apenas os seus proprios abastecimentos e o dono ve todos.

## Acceptance Criteria

1. Na pagina de detalhe da viagem (`/dashboard/viagens/[id]`), existe uma secao "Abastecimentos" com lista de todos os abastecimentos da viagem.
2. Cada item da lista exibe: data do abastecimento (formatada DD/MM/YYYY), litros (ex: 45,70 L), valor total (ex: R$ 294,83), preco por litro calculado (ex: R$ 6,45/L), nome do posto (se preenchido) e cidade/UF.
3. A lista exibe os abastecimentos em ordem cronologica decrescente (mais recente primeiro).
4. Quando nao ha abastecimentos registrados, a secao exibe estado vazio com mensagem orientativa (ex: "Nenhum abastecimento registrado. Toque em Registrar Abastecimento para adicionar.").
5. Cada item da lista possui um indicador visual (icone de camera ou thumbnail) quando ha foto do comprovante vinculada ao gasto.
6. Ao tocar no indicador de foto, abre a foto do comprovante em visualizacao fullscreen ou modal (reutilizar padrao existente de visualizacao de comprovantes).
7. Motorista (`role = motorista`) ve apenas os abastecimentos onde `gasto.motorista_id = id do usuario logado`.
8. Dono e admin (`role = dono` ou `role = admin`) veem todos os abastecimentos da viagem.
9. A secao exibe um resumo no topo com: total de abastecimentos, total de litros, custo total de combustivel da viagem.
10. A lista e responsiva e legivel em telas de 375px de largura (mobile), com texto minimo de 14px e itens de no minimo 60px de altura para facil toque.
11. Query usa JOIN `gasto + abastecimento_detalhe + foto_comprovante` filtrando por `viagem_id` e respeitando RLS existente.

### Query de referencia
```typescript
// lib/queries/combustivel-bi.ts
export async function getAbastecimentosPorViagem(
  supabase: SupabaseClient,
  viagemId: string
) {
  const { data, error } = await supabase
    .from('gasto')
    .select(`
      id,
      data,
      valor,
      km_registro,
      abastecimento_detalhe (
        litros,
        preco_litro,
        tipo_combustivel,
        km_odometro,
        posto,
        bandeira,
        cidade,
        estado
      ),
      foto_comprovante (
        storage_path,
        thumbnail_path
      )
    `)
    .eq('viagem_id', viagemId)
    .order('data', { ascending: false })

  return { data, error }
}
```
A RLS ja filtra automaticamente pelo role do usuario. Motorista vera apenas seus proprios gastos. Dono/admin verao todos.

### Calculo de preco/litro no cliente
```typescript
// preco/litro nao e armazenado em gasto, calcular na exibicao
const precoPorLitro = gasto.abastecimento_detalhe?.preco_litro
// ou calcular: gasto.valor / 100 / gasto.abastecimento_detalhe?.litros
```

### Formatacao de valores
- Litros: `${litros.toFixed(2).replace('.', ',')} L` → ex: "45,70 L"
- Valor: `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(valor / 100)` → ex: "R$ 294,83"
- Preco/litro: mesma formatacao com "/L" no final
- Data: `new Date(data).toLocaleDateString('pt-BR')` → ex: "29/03/2026"

### Foto do comprovante
O componente de visualizacao de fotos ja existe para a funcionalidade de comprovantes (Story 2.2). Verificar `components/comprovantes/` para reutilizar. Se houver `thumbnail_path`, usar thumbnail no item da lista e abrir `storage_path` em fullscreen.

### UX — publico 60+
- Texto de informacao secundaria (posto, cidade) em no minimo 13px
- Valor total em destaque (maior, negrito)
- Icone de camera claramente visivel (24px minimo)
- Nao usar cores sozinhas para indicar informacao
