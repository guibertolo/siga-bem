## Story
**As a** motorista ou dono,
**I want** ver a lista de todos os abastecimentos registrados em uma viagem,
**so that** eu possa acompanhar o historico de combustivel da viagem com data, litros, valor, preco/litro derivado, posto e foto do comprovante, sabendo que motorista ve apenas os seus e o dono ve todos.

## Acceptance Criteria

1. Na pagina de detalhe da viagem (`/dashboard/viagens/[id]`), existe uma secao "Abastecimentos" com lista de todos os abastecimentos da viagem.
2. Cada item da lista exibe: data (DD/MM/YYYY), litros (ex: 245,750 L), valor total (ex: R$ 1.584,00), preco por litro DERIVADO (calculado no frontend: `valor / 100 / litros`, ex: R$ 6,449/L), posto/local (se preenchido), UF do abastecimento.
3. O preco por litro NAO vem armazenado do banco — e calculado no frontend a partir de `gasto.valor` e `gasto.litros`.
4. A lista exibe os abastecimentos em ordem cronologica decrescente (mais recente primeiro).
5. Quando nao ha abastecimentos, a secao exibe estado vazio com mensagem orientativa (ex: "Nenhum abastecimento registrado. Use + Registrar Abastecimento para adicionar.").
6. Cada item possui indicador visual (icone de camera) quando ha foto do comprovante vinculada ao `gasto_id`. Ao tocar, abre foto em fullscreen ou modal (reutilizar padrao existente de comprovantes).
7. Motorista (`role = motorista`) ve apenas abastecimentos onde `gasto.motorista_id = id do usuario logado`. A RLS ja filtra automaticamente — nao requer logica extra no frontend.
8. Dono (`role = dono`) ve todos os abastecimentos da viagem. Admin tambem ve todos.
9. A secao exibe resumo no topo: total de abastecimentos, total de litros, custo total de combustivel da viagem.
10. Layout responsivo e legivel em 375px, texto minimo 14px, itens com minimo 60px de altura para facil toque.
11. Query filtra `gasto` por `viagem_id` E por categoria "Combustivel" (via `categoria_gasto.nome = 'Combustivel'` ou campo `litros IS NOT NULL` como indicador).

## Dev Notes

### Query de referencia
```typescript
// lib/queries/combustivel-queries.ts
export async function getAbastecimentosPorViagem(
  supabase: SupabaseClient,
  viagemId: string
) {
  const { data, error } = await supabase
    .from('gasto')
    .select(`
      id,
      data,
      valor,
      litros,
      tipo_combustivel,
      posto_local,
      uf_abastecimento,
      km_registro,
      foto_comprovante (
        storage_path,
        thumbnail_path
      )
    `)
    .eq('viagem_id', viagemId)
    .not('litros', 'is', null)   // apenas gastos de combustivel
    .order('data', { ascending: false })

  return { data, error }
}
```
A RLS ja filtra automaticamente. Motorista vera apenas seus proprios gastos.

### Calculo de preco/litro no frontend (NAO armazenado)
```typescript
// Calcular no componente, nao vem do banco
const precoLitro = (gasto.valor / 100) / gasto.litros
// Formatar: Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(precoLitro) + '/L'
// Ex: "R$ 6,449/L"
```

### Formatacao de valores
- Litros: `${gasto.litros.toFixed(3).replace('.', ',')} L` → ex: "245,750 L"
- Valor: `Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(gasto.valor / 100)` → ex: "R$ 1.584,00"
- Preco/litro: mesma formatacao de currency + "/L"
- Data: `new Date(gasto.data).toLocaleDateString('pt-BR')`

### Foto do comprovante
O componente de visualizacao de fotos ja existe para a funcionalidade de comprovantes (Stories 2.x). Verificar `components/comprovantes/` para reutilizar o visualizador fullscreen/modal.

### UX — publico 60+
- Valor total em destaque (maior, negrito)
- Preco/litro e litros em secundario (menor)
- Icone de camera claramente visivel (24px minimo)
- Nao usar cores sozinhas para indicar informacao
