## Story
**As a** dono de frota,
**I want** um relatorio completo de um caminhao em um periodo, com custo por km, margem e comparativo com outros caminhoes,
**so that** eu saiba quais caminhoes estao gerando lucro e quais estao pesando no caixa.

## Epic
Epic 23 (REL) — Relatorios e Upload da Chamada.

## Acceptance Criteria

1. RPC `relatorio_caminhao_periodo(p_caminhao_id uuid, p_inicio date, p_fim date) RETURNS jsonb` criada em `supabase/migrations/` com `SECURITY INVOKER`.
2. A RPC retorna um objeto JSONB com a estrutura:
   - `header`: `{ caminhao_placa, caminhao_modelo, empresa_nome, periodo_inicio, periodo_fim, total_viagens, km_total_calculado, receita_total_centavos, custo_total_centavos, custo_por_km_centavos, margem_absoluta_centavos, margem_percentual }` — onde `km_total_calculado = SUM(km_chegada - km_saida)` apenas para viagens com ambos os km nao-null; `custo_por_km_centavos = ROUND(custo_total / NULLIF(km_total_calculado, 0))::INTEGER`; `margem_percentual = ROUND((receita - custo) * 100.0 / NULLIF(receita, 0), 1)`.
   - `viagens`: array de `{ id, data_saida, data_chegada_real, origem, destino, km_calculado, valor_total_centavos, status, motorista_nome }` — `km_calculado = km_chegada - km_saida` (null-safe); sem `chamada_url_signed` (tabela nao existe no schema atual).
   - `motoristas_que_rodaram`: array de `{ nome, cpf_mascarado, total_viagens, km_total_calculado }` distintos no periodo.
   - `custos_diretos`: `{ combustivel_centavos, manutencao_centavos, pedagio_centavos }` — extraidos de `gasto JOIN categoria_gasto ON categoria_gasto.id = gasto.categoria_id WHERE gasto.caminhao_id = p_caminhao_id AND categoria_gasto.nome ILIKE 'Combustivel'` (analogamente para 'Manutencao' e 'Pedagio'). Nome e texto seed, nao enum.
   - `comparativo_frota`: `{ posicao_receita, posicao_margem, total_caminhoes }` — ranking deste caminhao vs outros da empresa por `SUM(valor_total)` e por margem no periodo.
   - `dias_em_rota`: integer (COUNT de datas distintas em `data_saida` no periodo).
   - `dias_parado`: integer (dias corridos no periodo menos `dias_em_rota`).
3. Server Action `gerarRelatorioCaminhao` em `app/(dashboard)/relatorios/caminhao/actions.ts` que: (a) valida que `caminhao_id` pertence a `empresa_id` da sessao (`caminhao.empresa_id`); (b) chama a RPC; (c) retorna dados tipados.
4. Tela `app/(dashboard)/relatorios/caminhao/[id]/page.tsx` (Server Component) renderiza dados em cards empilhados mobile-first, com mesma estrutura visual da tela de motorista (23.5), adaptada para caminhao:
   - Header com placa, modelo, periodo e KPIs principais (km calculado, receita, custo/km, margem%).
   - Cards de viagens com nome do motorista de cada viagem.
   - Card de custos diretos por categoria (combustivel, manutencao, pedagio).
   - Card de comparativo com outros caminhoes da frota.
   - Nao exibir card de alertas nesta versao (bloqueado por 18.1).
   - Botoes "Baixar PDF" e "Baixar Excel" no topo (funcionais apos 23.7/23.8).
5. `@media print` converte para formato A4 com nome da empresa em texto (sem logo — campo nao existe). Mesma abordagem do 23.5.
6. Defesa-em-profundidade: server action rejeita com 403 se `caminhao_id` nao pertencer a empresa da sessao.
7. Calculo de `custo_por_km_centavos` em centavos/km: `ROUND(custo_total / NULLIF(km_total_calculado, 0))::INTEGER`. Exibir como R$/km com 2 casas decimais na UI.
8. Calcular `margem_percentual` como `ROUND((receita - custos_diretos) * 100.0 / NULLIF(receita, 0), 1)`. Exibir com sinal (ex: "+12,3%" verde, "-3,1%" vermelho). Null quando receita = 0.
9. Indice `viagem(caminhao_id, data_saida)` (`idx_viagem_caminhao_data` criado em 23.1) deve estar comentado no SQL da RPC: `-- uses index idx_viagem_caminhao_data`.
10. Testes de integracao com Supabase local validam isolamento multi-tenant (caminhao de empresa B nao visivel para empresa A).

**Categorias de gasto — seeds confirmados (migration `20260328180500`:linhas 53-64):**
```
'Pedagio'     (id gerado, nome texto)
'Combustivel' (id gerado, nome texto)
'Manutencao'  (id gerado, nome texto)
```
Nao existe enum — filtrar SEMPRE por nome com `ILIKE`. Query SQL:
```sql
SUM(CASE WHEN cg.nome ILIKE 'Combustivel' THEN g.valor ELSE 0 END) AS combustivel_centavos,
SUM(CASE WHEN cg.nome ILIKE 'Manutencao'  THEN g.valor ELSE 0 END) AS manutencao_centavos,
SUM(CASE WHEN cg.nome ILIKE 'Pedagio'     THEN g.valor ELSE 0 END) AS pedagio_centavos
FROM gasto g
JOIN categoria_gasto cg ON cg.id = g.categoria_id
WHERE g.caminhao_id = p_caminhao_id
  AND g.data BETWEEN p_inicio AND p_fim
```

**Calculos financeiros — sempre em centavos INTEGER:**
```sql
-- Custo por km (centavos por km, arredondar)
ROUND(SUM(custo_total)::numeric / NULLIF(SUM(km_calculado), 0))::INTEGER AS custo_por_km_centavos

-- Margem percentual (1 casa decimal)
ROUND((receita - custos) * 100.0 / NULLIF(receita, 0), 1) AS margem_percentual
```
