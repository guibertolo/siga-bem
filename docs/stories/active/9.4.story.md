## Acceptance Criteria

1. Arquivo `lib/copilot/tools/motoristas-cnh-vencendo.ts` criado:
   - **schema (Zod):** `{ dias?: number }`. Default = 30. Max = 365.
   - **execute:** query em `motorista` com `.eq('status', 'ativo')`, `.gte('cnh_validade', hoje)`, `.lte('cnh_validade', hoje + dias)`, `.in('empresa_id', empresaIds)`, `.order('cnh_validade')`, `.limit(MAX_TOOL_ROWS)`. Retorna `{ motoristas: [{ id, nome, cnh_numero, cnh_categoria, cnh_validade, dias_ate_vencer }] }`.
   - Inclui tambem motoristas com CNH ja vencida? **Sim** — adicionar motoristas com `cnh_validade < hoje` ao resultado com `dias_ate_vencer < 0`, ordenados do mais vencido para o mais recente.
2. Arquivo `lib/copilot/tools/resumo-desempenho-periodo.ts` criado:
   - **schema (Zod):** `{ periodo: string }`
   - **execute:** agrega num unico objeto: `{ periodo: {start, end}, qtd_viagens, receita_total_centavos, gasto_total_centavos, lucro_centavos, top_3_categorias: [{ nome, total_centavos }], melhor_viagem: { id, origem, destino, margem_percentual } | null, pior_viagem: { id, origem, destino, margem_percentual } | null }`.
   - Excluir viagens canceladas.
   - Se nao houver viagens no periodo, retornar zeros e `melhor_viagem/pior_viagem = null` (sem erro).
3. Ambas respeitam RLS via `createClient()` SSR + `.in('empresa_id', empresaIds)`.
4. Ambas aplicam cap `MAX_TOOL_ROWS` onde aplicavel.
5. Valores em centavos. Datas ISO. `dias_ate_vencer` e `number` (pode ser negativo).
6. Testes de integracao em `__tests__/copilot/tools/cnh-vencendo.test.ts` (incluindo caso de CNH ja vencida e caso sem motoristas) e `__tests__/copilot/tools/resumo-desempenho.test.ts` (incluindo caso de periodo vazio).
7. Try/catch + logger.error.
8. `npm run lint && typecheck && test && build` passam.

## Dev Notes

### CNH vencida
Embora a pergunta-farol seja "CNH vencendo nos proximos 30 dias", incluir vencidas e uma informacao criticamente util para o dono. Sem quebrar contrato, retornamos vencidas + a vencer na mesma lista, ordenadas pelo campo `dias_ate_vencer` crescente (vencidas ficam no topo com valores negativos).

### Sem tabela de documento
Reforca-se que o FrotaViva **nao tem** tabela de documento generica. Esta tool cobre APENAS CNH. Outros documentos (CRLV, seguro, etc.) ficam fora do MVP (spec §1).

### Periodo padrao do resumo
Se o LLM nao passar periodo, o Zod schema nao aceita — `periodo` e obrigatorio. O LLM deve inferir do prompt do usuario ("ultima semana" -> passa "ultima semana"). Se o usuario falar "me da um resumo" sem periodo, o LLM pergunta antes de chamar a tool — isso eh tratado via system prompt na 9.5.
