## Story
**As a** usuario do FrotaViva (motorista ou dono/admin),
**I want** uma pagina de perfil onde posso ver meus dados e trocar minha senha,
**so that** eu possa manter minha conta segura sem depender do dono para operacoes de senha.

## Acceptance Criteria

1. Existe a rota `/perfil` com uma page component em `app/(dashboard)/perfil/page.tsx`.
2. O link "Meu Perfil" aparece no sidebar da aplicacao (tanto desktop quanto mobile), posicionado abaixo de todos os itens de navegacao e acima do botao "Sair".
3. A pagina exibe os dados do usuario autenticado em campos readonly: Nome, Email, Telefone e Role (exibido como texto amigavel: "Motorista", "Proprietario", "Administrador").
4. Para usuario com role `motorista`: TODOS os campos de dados (nome, email, telefone, role) sao exibidos como texto readonly, sem inputs editaveis. Apenas a secao de troca de senha e interativa.
5. Para usuario com role `dono` ou `admin`: os campos Nome e Telefone sao inputs editaveis com botao "Salvar Alteracoes". Email e Role permanecem readonly.
6. A secao "Alterar Senha" e exibida para TODOS os roles e contem: campo "Senha atual", campo "Nova senha", campo "Confirmar nova senha" e botao "Alterar Senha".
7. A validacao da nova senha exige minimo de 8 caracteres. Mensagem de erro: "A senha deve ter pelo menos 8 caracteres".
8. A validacao confirma que "Nova senha" e "Confirmar nova senha" sao identicas. Mensagem de erro: "As senhas nao coincidem".
9. Ao clicar "Alterar Senha", a server action chama `supabase.auth.updateUser({ password: novaSenha })` com o cliente de sessao (NAO o admin client).
10. Se a senha atual for incorreta, exibe mensagem de erro: "Senha atual incorreta".
11. Se a troca de senha for bem-sucedida, exibe mensagem de sucesso: "Senha alterada com sucesso!" e limpa os campos da secao de senha.
12. Para dono/admin, a server action de edicao de perfil atualiza `nome` e `telefone` na tabela `usuario` via UPDATE WHERE `auth_id = auth.uid()`. Email e role NAO podem ser alterados via esta action.
13. O campo telefone aceita formato livre (texto), sem mascara obrigatoria.
14. A pagina carrega os dados do usuario via Server Component (sem loading flash).
15. Todos os campos e botoes funcionam em viewport de 375px (mobile-first, fonte minima 16px nos inputs).

## Dev Notes

### Regra de Negocio Critica
O motorista NAO pode alterar nome, email ou telefone. Apenas o dono/admin pode alterar dados cadastrais. Esta regra foi definida pelo stakeholder e e inegociavel: "Imagina o problema o motorista trocar de nome durante uma viagem e acabar se perdendo os dados."

A restricao deve ser aplicada em duas camadas:
1. **UI**: campos readonly para motorista (AC 4)
2. **Server action**: `atualizarPerfil` valida role antes de executar UPDATE (AC 12)

### Server Component para Dados
A page deve ser um Server Component para carregar dados sem flash:
```typescript
export default async function PerfilPage() {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();
  const { data: usuario } = await supabase
    .from('usuario')
    .select('nome, email, telefone, role')
    .eq('auth_id', user!.id)
    .single();
  // passar como props para componentes client
}
```

### Troca de Senha com supabase.auth.updateUser
O `supabase.auth.updateUser({ password })` do cliente de sessao valida automaticamente que o usuario esta autenticado. NAO requer a senha atual explicitamente -- o Supabase valida a sessao ativa como prova de autenticidade.

Para exigir a senha atual como camada extra de seguranca (AC 10), fazer:
1. Chamar `supabase.auth.signInWithPassword({ email: usuario.email, password: senhaAtual })` primeiro
2. Se retornar erro, rejeitar a troca
3. Se bem-sucedido, chamar `supabase.auth.updateUser({ password: novaSenha })`

### Sidebar: Posicionamento do Link
No layout desktop (`app/(dashboard)/layout.tsx`) e no `MobileSidebar.tsx`, o link "Meu Perfil" deve estar entre o ultimo item de navegacao e o botao "Sair". Verificar o HTML atual para identificar o ponto de insercao correto.

### Role Display
Mapeamento de role para texto amigavel:
```typescript
const roleLabel = { dono: 'Proprietario', motorista: 'Motorista', admin: 'Administrador' };
```

### Campos Editaveis para Dono/Admin
Para dono e admin, usar `<input>` para nome e telefone com estado controlado. Para motorista, usar `<p>` ou `<span>` styled para parecer campo mas sem interatividade.
