## Story
**As a** dono de frota,
**I want** baixar o relatorio de motorista ou caminhao em PDF com logo da empresa, tabela de viagens e area de assinatura,
**so that** eu possa imprimir e assinar com o motorista no acerto semanal, tendo prova documental do acordo.

## Epic
Epic 23 (REL) — Relatorios e Upload da Chamada.

## Acceptance Criteria

1. Rota `app/(dashboard)/relatorios/[tipo]/[id]/pdf/route.ts` criada com `export const runtime = 'nodejs'` explicito (nunca Edge — `@react-pdf/renderer` nao roda em Edge runtime). Usar o route group `(dashboard)` que ja existe no projeto.
2. A rota valida sessao autenticada e que `id` pertence a `empresa_id` da sessao antes de gerar qualquer dado. Retorna 401 (sem sessao) ou 403 (id nao autorizado).
3. Para `tipo=motorista`: chama dados equivalentes ao 23.5 (pode reusar a server action `gerarRelatorioMotorista`). Para `tipo=caminhao`: chama dados equivalentes ao 23.6.
4. Para total de viagens <= 200: gera PDF sincronamente via `renderToStream` do `@react-pdf/renderer` e retorna stream com `Content-Type: application/pdf` e `Content-Disposition: attachment; filename="relatorio-{tipo}-{nome}-{periodo}.pdf"`.
5. Para total de viagens > 200: retorna resposta JSON `{ status: 'queued', message: 'Relatorio grande, processando...' }` com HTTP 202. Implementacao do job assincrono e stub (P2 — story futura 23.x). Logar o evento no Sentry com `Sentry.captureMessage`.
6. Componentes PDF criados em `src/components/pdf/`:
   - `RelatorioMotoristaPdf.tsx` — layout do relatorio de motorista
   - `RelatorioCaminhaoPdf.tsx` — layout do relatorio de caminhao
7. Layout do PDF inclui obrigatoriamente:
   - Cabecalho: nome da empresa em texto. Logo omitida nesta story (omitir `empresa.logo_url` — usar apenas texto com o nome da empresa).
   - Tabela de viagens com colunas: Data, Origem/Destino, KM, Valor Frete, Comissao, Status
   - Totais ao final da tabela
   - Secao "Chamadas Fotografadas" ao final do documento como PAGINAS SEPARADAS (uma por chamada), usando `<Page>` novo para cada imagem de chamada
   - Area de assinatura no final: duas linhas "Dono: ___________" e "Motorista: ___________" com espaco para rubrica
   - Rodape com data de geracao e "FrotaViva — frotaviva.com.br"
8. Botao "Baixar PDF" nas telas 23.5 e 23.6 conectado a esta rota via `<a href="/relatorios/{tipo}/{id}/pdf" download>`.
9. Testes manuais documentados: gerar PDF com 1, 10, 100 e 200 viagens mock e verificar que nao ha timeout nem erro de memoria.
10. Nao instalar nova dependencia de PDF — `@react-pdf/renderer` 4.3.2 ja esta no `package.json`.

## Dev Notes

**Dependencias obrigatorias:** Stories 23.5 e 23.6 (server actions de dados). Story 23.4 (telas com os botoes de download). Story 23.1 BLOCKING — a tabela `foto_chamada` deve existir para que as chamadas fotografadas sejam incluidas no PDF (a rota pode ser implementada sem 23.1, mas a secao de chamadas ficara vazia ate 23.1 estar concluida).

**CRITICO — runtime Node obrigatorio:**
```typescript
// app/(dashboard)/relatorios/[tipo]/[id]/pdf/route.ts
export const runtime = 'nodejs'; // NAO remover — @react-pdf nao funciona em Edge
export const dynamic = 'force-dynamic';
```
Sem essa linha o Vercel direciona para Edge Function e `@react-pdf/renderer` crasha com erro de modulo.

**Estrutura de rota:** usar `app/(dashboard)/relatorios/[tipo]/[id]/pdf/route.ts` — o route group `(dashboard)` ja existe e e o correto para o projeto.

**API do @react-pdf/renderer 4.3.2 — verificado no node_modules:** `renderToStream` esta disponivel e e o metodo correto para uso em Node.js route handler. Nao usar `renderToBuffer` (retorna stream tambem, renomeado internamente — preferir `renderToStream` que e mais explicito).

**Padrao de renderizacao do @react-pdf/renderer:**
```typescript
import { renderToStream } from '@react-pdf/renderer';
import { RelatorioMotoristaPdf } from '@/components/pdf/RelatorioMotoristaPdf';

const stream = await renderToStream(<RelatorioMotoristaPdf data={dados} />);
return new Response(stream as unknown as ReadableStream, {
  headers: {
    'Content-Type': 'application/pdf',
    'Content-Disposition': `attachment; filename="relatorio-motorista-${nome}-${periodo}.pdf"`,
  },
});
```

**Threshold de 200 viagens:**
```typescript
const totalViagens = dados.viagens.length;
if (totalViagens > 200) {
  Sentry.captureMessage('Relatorio PDF grande solicitado', {
    extra: { tipo, id, totalViagens },
  });
  return Response.json({ status: 'queued', message: 'Relatorio grande, processando...' }, { status: 202 });
}
```
O job assincrono real e P2 (story futura). Por ora, apenas logar e retornar 202.

**Componentes PDF — estrutura basica:**
```tsx
import { Document, Page, Text, View, Image, StyleSheet } from '@react-pdf/renderer';

const styles = StyleSheet.create({
  page: { flexDirection: 'column', padding: 30, fontFamily: 'Helvetica' },
  header: { marginBottom: 20 },
  tabela: { width: '100%', borderStyle: 'solid', borderWidth: 1 },
  // ...
});

export function RelatorioMotoristaPdf({ data }: { data: RelatorioMotoristaResult }) {
  return (
    <Document>
      <Page size="A4" style={styles.page}>
        {/* Cabecalho */}
        {/* Tabela de viagens */}
        {/* Totais */}
        {/* Area de assinatura */}
      </Page>
      {/* Paginas de chamadas — uma <Page> por chamada */}
      {data.viagens.flatMap(v => v.chamada_url_signed ? [v] : []).map((v, i) => (
        <Page key={i} size="A4" style={styles.page}>
          <Text>Chamada - {v.data_saida}</Text>
          <Image src={v.chamada_url_signed!} />
        </Page>
      ))}
    </Document>
  );
}
```
