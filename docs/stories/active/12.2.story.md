## Story
**Como** usuario do FrotaViva no publico 55+,
**Quero** que acoes como ver comprovante de gasto, ver credenciais de motorista e convidar novo usuario acontecam em paginas ou cards inline, nao em modais que cobrem a tela inteira,
**Para que** eu possa voltar com o botao "voltar" do navegador se me confundir e nao sinta que perdi o contexto da pagina onde estava.

## Acceptance Criteria

1. `ReceiptModal` (comprovante de gasto) e convertido para pagina dedicada `/gastos/[id]/comprovante` OU card inline dentro de `GastoList` ou `/gastos/[id]`. A escolha e do @dev com base em qual navega melhor, mas nao pode continuar como modal `fixed-inset-0`.
2. `CredenciaisModal` (credenciais geradas de motorista) e convertido para pagina dedicada `/motoristas/[id]/credenciais` OU card inline na pagina de detalhe do motorista. Mesma regra.
3. `InviteModal` (convidar usuario para empresa) e convertido para pagina dedicada `/empresa/convidar` OU card inline na pagina de empresa. Mesma regra.
4. Os 3 modais originais sao deletados do codigo. Uma busca por `ReceiptModal`, `CredenciaisModal`, `InviteModal` nao encontra definicoes nem imports.
5. Os fluxos funcionam end-to-end em mobile e desktop apos conversao:
   - Ver comprovante: abrir gasto, clicar em "ver comprovante", ver comprovante, voltar.
   - Ver credenciais: apos criar motorista, ver tela com credenciais temporarias, copiar, voltar.
   - Convidar usuario: ir em empresa, clicar em convidar, preencher email, enviar, ver confirmacao.
6. Se a conversao for para pagina dedicada, o browser back button funciona (volta para a lista de origem).
7. Nenhum uso de `createPortal` para overlay sobre a tela inteira nesses 3 fluxos. O `OnboardingBar` e outros componentes nao sao afetados por esta story (ver story 11.x do backlog P3).
8. `npm run typecheck && npm run lint && npm run build` passam.

## Dev Notes

- Recomendacao: quando o conteudo e auto-contido (ex: credenciais geradas so uma vez), pagina dedicada e melhor porque preserva a URL e permite refresh sem perder.
- Quando o conteudo e detalhe de um item ja em lista (ex: comprovante de gasto), card inline expansivel na linha do gasto funciona bem.
- CredenciaisModal tem uma particularidade: as credenciais nao ficam salvas em clear (so o hash), entao a pagina dedicada so funciona se chamada imediatamente apos o create com os valores em estado. Se tentar acessar depois nao tem dado. Decidir com cuidado.
- NAO afetar o comportamento transacional. Se o backend gera convite via email, continuar gerando. Se gera credencial em sessao, manter.
- Regra do usuario: sem popups. Esta story + 12.1 cobrem a maioria dos popups do produto.
