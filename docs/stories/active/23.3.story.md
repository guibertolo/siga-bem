## Story
**As a** dono de frota,
**I want** ver a foto da chamada na ficha da viagem ao lado dos comprovantes,
**so that** eu possa conferir o documento do motorista sem precisar ligar para ele ou pedir para reenviar.

## Epic
Epic 23 — REL (Relatorios e Upload da Chamada).

## Acceptance Criteria

1. Na pagina de detalhe da viagem (`/viagens/[id]`), a chamada aparece como miniatura 80x80px clicavel:
   - Posicionada ao lado dos comprovantes de gastos, na mesma area visual
   - Label "Chamada" abaixo da miniatura
   - Borda arredondada consistente com o design system existente

2. Clique na miniatura abre visualizacao em tela cheia:
   - Imagem ocupa toda a tela (`position: fixed`, `inset: 0`, `z-index` alto) com `style` inline para `100vw/100vh` (ver `feedback_fullscreen_elements.md`)
   - Fundo preto semitransparente
   - Suporte a zoom via pinch-to-zoom no mobile: escala de 1x a 4x; double-tap reseta para 1x. Pode usar a lib `react-zoom-pan-pinch` (~50KB) se o dev preferir em vez de implementar manualmente com eventos de touch.
   - Botoes de zoom `+` e `-` para desktop (incrementos de 25%)
   - Botao `X` grande e visivel no canto superior direito para fechar (minimo 48px)
   - Botao "Baixar" (icone de download) — dispara download da imagem com nome `chamada-{viagem_id}.webp`

3. Dono e admin veem botao "Substituir chamada" na ficha da viagem:
   - Botao secundario (nao destrutivo visualmente)
   - Ao clicar, abre o mesmo fluxo de upload da Story 23.2 (reusar componente)
   - Apos substituicao, a foto antiga permanece no Storage (imutabilidade auditavel) mas a nova torna-se a exibida

4. Motorista nao ve botao "Substituir chamada" nem opcao de remover:
   - Verificar `usuario.role` no componente antes de renderizar o botao
   - Zero opcao de modificacao visivel para motorista

5. Quando ha mais de uma chamada (dono substituiu):
   - Exibir a foto mais recente como miniatura principal
   - Abaixo da miniatura, label pequeno `+N foto anterior` onde N e o numero de fotos anteriores
   - Clicar no label abre lista simples com todas as fotos em ordem decrescente de data

6. Zero-state — viagem sem chamada:
   - Se o motorista pode adicionar (viagem `em_andamento` e role `motorista`): exibir card "Chamada pendente" com CTA "Adicionar agora" que aciona o fluxo de upload (Story 23.2)
   - Se o dono/gestor visualiza viagem sem chamada: exibir card "Chamada nao enviada" com botao "Adicionar chamada"
   - Se a viagem esta concluida e sem chamada: exibir texto "Chamada nao registrada" sem CTA (nao e mais possivel adicionar pelo motorista)

7. Signed URL para exibicao:
   - Buscar signed URL com expiry de 1 hora via server action `getChamadaSignedUrl(chamadaId: string)`
   - Nao expor `storage_path` direto no cliente

8. `npm run lint && npm run typecheck` passam sem novos erros.

## Dev Notes

**Origem:** Addendum 2026-04-14, item #121. Epic L, P1, M/S. Depende de Story 23.2 (chamada pode ser adicionada e existir no banco).

**Pagina alvo:** `app/(dashboard)/viagens/[id]/page.tsx` e componentes filhos. Verificar a estrutura atual da pagina de detalhe antes de adicionar o componente de chamada.

**Componente de miniatura:** criar `components/viagens/ChamadaMiniatura.tsx`. Props:
- `chamadas: FotoChamada[]` — lista em ordem decrescente de `uploaded_at`
- `viagemId: string`
- `viagemStatus: 'planejada' | 'em_andamento' | 'concluida' | 'cancelada'`
- `usuarioRole: string`

**Componente de tela cheia:** criar `components/viagens/ChamadaViewer.tsx`. Implementar como elemento fixo montado no DOM (nao modal com `fixed inset-0` via portal — regra `feedback_no_popups.md` do usuario se aplica a modais de fluxo; viewer de imagem em tela cheia e aceitavel pois e navegacao, nao interrupcao de fluxo). Usar `style` inline para `position: fixed; inset: 0; width: 100vw; height: 100vh` (regra `feedback_fullscreen_elements.md`).

**Zoom no mobile:** usar CSS `touch-action: pinch-zoom` + `transform: scale()` controlado por estado. Escala minima 1x, maxima 4x, double-tap reseta para 1x. O dev pode optar por `react-zoom-pan-pinch` (~50KB) em vez da implementacao manual com eventos de touch — ambas as abordagens sao aceitas.

**Download:** usar `fetch` para obter a imagem via signed URL e criar um `<a download>` temporario com `URL.createObjectURL`. Nao abrir em nova aba.

**Server action nova:** adicionar em `app/(dashboard)/viagens/chamada-actions.ts` (criado na Story 23.2):
- `getChamadaSignedUrl(chamadaId: string): Promise<{url: string} | {error: string}>`
- `listChamadasViagem(viagemId: string): Promise<ChamadasListResult>`
- `uploadChamadaDono(formData: FormData): Promise<ChamadaActionResult>` — versao sem restricao de status para dono/admin (nao restrita a `planejada` ou `em_andamento`). Este AC e critico e deve aparecer na lista de subtasks, nao apenas nas Dev Notes.

**Posicionamento na pagina:** a pagina `/viagens/[id]` exibe comprovantes de gastos. A chamada deve aparecer na mesma regiao visual, com label diferenciador. Verificar o componente que renderiza os comprovantes e adicionar a chamada ao lado ou logo abaixo.

**Zero-state contextual:** tres estados distintos (AC: 6). Nao usar um estado generico — o texto e a acao devem refletir exatamente o que o usuario pode fazer naquele momento.

**Label `+N foto anterior`:** apenas quando `chamadas.length > 1`. Para `chamadas.length === 1`, nao exibir o label.

**Paths de referencia:**
- Pagina de detalhe: `app/(dashboard)/viagens/[id]/page.tsx`
- Action existente (modelo): `app/(dashboard)/viagens/chamada-actions.ts` (criado em 23.2)
- Tipos: `types/foto-chamada.ts` (criado em 23.2)
- Regra fullscreen inline: `feedback_fullscreen_elements.md` — sempre `style` inline para 100vw/100vh
- Signed URL pattern: `app/(dashboard)/gastos/comprovante-actions.ts` funcao `listComprovantes` (expiry 3600)

**Publico 55+:** botao X para fechar DEVE ser grande e no canto superior direito. Icone sozinho nao e suficiente — incluir texto "Fechar" ou tooltip visivel. Tamanho minimo 48px de area clicavel.
