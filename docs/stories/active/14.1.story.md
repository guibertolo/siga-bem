## Story
**Como** desenvolvedor do FrotaViva,
**Quero** que o CI esteja verde, que zero erros de typecheck e lint existam, e que um hook de pre-commit impeca que novos erros entrem,
**Para que** qualquer refatoracao futura (Epics B, E) parta de uma base saudavel e qualquer regressao seja detectada antes de chegar no commit.

## Acceptance Criteria

1. Os 2 erros de typecheck em `VeiculoForm.tsx` sao corrigidos. `npm run typecheck` retorna zero erros.
2. Os 9 erros de lint identificados no baseline FIRE sao corrigidos. `npm run lint` retorna zero erros (warnings sao aceitos, mas erros nao).
3. `npm run build` completa sem erro.
4. Husky esta instalado e configurado com hook `pre-commit` que roda `npm run typecheck && npm run lint && npm test -- --passWithNoTests`.
5. Tentar commitar um arquivo com erro de typecheck (proposital) faz o hook bloquear o commit.
6. O CI no GitHub Actions esta funcional: push para main dispara o workflow e todos os steps passam (lint, typecheck, build). Se `npm test` falhar por falta de testes, usar flag `--passWithNoTests` no CI tambem.
7. Branch protection em `main` esta ativada: merge so com CI verde.
8. O warning de lint `react-hooks/exhaustive-deps` em `GastoForm.tsx` e tratado (seja corrigindo a dependencia, seja adicionando ignore comentado com justificativa).
9. Os 4 warnings de `<img>` vs `<Image />` sao resolvidos: ou trocados por `next/image` ou marcados com `// eslint-disable-next-line @next/next/no-img-element` com justificativa.
