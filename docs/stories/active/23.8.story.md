## Story
**As a** contador ou dono de frota,
**I want** baixar o relatorio em Excel com abas separadas para viagens, despesas por categoria dedutivel, abastecimentos e resumo de KPIs,
**so that** eu possa importar direto no meu ERP ou entregar para o contador sem retrabalho de formatacao.

## Epic
Epic 23 (REL) — Relatorios e Upload da Chamada.

## Acceptance Criteria

1. Pacote `write-excel-file` instalado via `npm install write-excel-file` e adicionado ao `package.json`. Bundle de aproximadamente 200KB — confirmar com `npm run build`.
2. Rota `app/(dashboard)/relatorios/[tipo]/[id]/xlsx/route.ts` criada com `export const runtime = 'nodejs'` explicito. Usar o route group `(dashboard)` consistente com o resto do projeto e com a story 23.7.
3. A rota valida sessao autenticada e que `id` pertence a `empresa_id` da sessao. Retorna 401 ou 403 conforme o caso.
4. O arquivo Excel gerado contem 4 abas na ordem:
   - **Viagens** — uma linha por viagem: Data Saida, Data Chegada, Origem, Destino, KM, Valor Frete (R$), Comissao (R$), Status, Motorista/Caminhao (conforme tipo do relatorio)
   - **Despesas** — uma linha por gasto listando TODAS as categorias sem classificacao de deducao (nao filtrar por "dedutivel" — listar todas). Colunas: Categoria, Descricao, Data, Valor (R$), KM no momento. Ordem: data crescente
   - **Abastecimentos** — uma linha por abastecimento. "Abastecimento" no FrotaViva e o registro `gasto` com `categoria='Combustivel'`. Colunas: Data, Posto (`posto_local` se disponivel), Litros (`litros`), Valor (R$), KM Abastecido (`km_registro`), KM/L (calculado como `(km_registro_atual - km_registro_anterior) / litros_atual`, por caminhao em ordem cronologica)
   - **Resumo** — bloco de KPIs: Total de Viagens, KM Total, Receita Bruta (R$), Custo Total (R$), Custo/KM (R$/km), Margem Bruta (R$), Margem (%), Periodo, Nome Motorista/Caminhao, Empresa, Data de Geracao
5. Formato brasileiro em todas as abas: separador decimal virgula, datas em DD/MM/AAAA, valores monetarios como numero (nao texto) com 2 casas decimais para o Excel aplicar formatacao de moeda.
6. Primeira linha congelada (`freeze panes`) em todas as abas via `{ stickyRowsCount: 1 }` (API de `write-excel-file`). Cabecalhos em negrito: usar o campo `fontWeight: 'bold'` disponivel na API de `write-excel-file` para a linha de header.
7. Largura de colunas ajustada automaticamente pelo conteudo (auto-width baseado no conteudo das celulas).
8. Response com `Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet` e `Content-Disposition: attachment; filename="relatorio-{tipo}-{nome}-{periodo}.xlsx"`.
9. Botao "Baixar Excel" nas telas 23.5 e 23.6 conectado a esta rota via `<a href="/relatorios/{tipo}/{id}/xlsx" download>`.
10. A aba Despesas lista TODAS as categorias de gasto sem filtro de "dedutivel" (o conceito de deducao e removido desta story). O contador ou usuario decide o que e dedutivel na sua ferramenta.

// Schema para aba Viagens
const viagensSchema = [

  { column: 'Origem', type: String, value: (v: Viagem) => v.origem },

const buffer = await writeXlsxFile([dados.viagens, dados.despesas, dados.abastecimentos, resumoData], {

**Auto-width:** a lib `write-excel-file` calcula largura automaticamente com base no conteudo quando `columnWidth` nao e especificado — nao e necessario helper manual.

**Negrito nos cabecalhos:** a lib `write-excel-file` suporta `fontWeight: 'bold'` nativo no schema. Nao precisa de fork nem dependencia adicional.

**Formato de datas no Excel:**
Datas devem ser STRING no formato DD/MM/AAAA (nao Date object) para garantir que o Excel brasileiro leia corretamente sem conversao de timezone.
```typescript
function formatDateBR(dateStr: string): string {
  // dateStr = 'YYYY-MM-DD'
  const [y, m, d] = dateStr.split('-');
  return `${d}/${m}/${y}`;
}
```

**Abastecimentos na aba Abastecimentos:**
"Abastecimento" = registro `gasto` com `categoria = 'Combustivel'` (ou valor equivalente no enum). As colunas especificas sao `litros`, `posto_local`, `km_registro` — verificar se esses campos existem em `gasto` antes de implementar.
