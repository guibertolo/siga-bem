# Story 22.5: Signed URL com TTL reduzido em listagens

## Story
**As a** responsavel pela seguranca do FrotaViva,
**I want** que URLs assinadas de arquivos (comprovantes, documentos) expirem rapidamente em contextos de listagem,
**so that** links capturados em logs ou cache de proxies nao permitam acesso prolongado a arquivos sensiveis.

## Epic
Epic 22 — Infra Hardening.

## Origem
Addendum 2026-04-14, item #5

## Acceptance Criteria

1. A constante `SIGNED_URL_EXPIRY = 3600` em `app/(dashboard)/gastos/comprovante-actions.ts` linha 13 e substituida por duas constantes separadas:
   - `SIGNED_URL_EXPIRY_LIST = 300` (5 minutos, para contextos de listagem)
   - `SIGNED_URL_EXPIRY_DETAIL = 3600` (1 hora, para contextos de detalhe)

2. A chamada `createSignedUrl` em `comprovante-actions.ts` linha 262, dentro da funcao `listComprovantes()`, passa a usar `SIGNED_URL_EXPIRY_LIST` (300s). Esta funcao itera `comprovantes[]` via `Promise.all` e serve o contexto de listagem de comprovantes de um gasto.

3. A chamada `createSignedUrl` em `comprovante-actions.ts` linha 140, dentro da funcao `uploadComprovante()`, mantem `SIGNED_URL_EXPIRY_DETAIL` (3600s). Esta chamada ocorre uma unica vez apos o upload para gravar `foto_url` no gasto — contexto de detalhe/registro, nao listagem.

4. A chamada `createSignedUrl` em `app/(dashboard)/viagens/[id]/actions.ts` linha 280, com TTL hardcoded `3600`, permanece inalterada. Esta funcao retorna gastos de uma viagem especifica (pagina de detalhe individual) e o TTL de 1 hora e adequado para esse contexto.

5. A interface publica de `uploadComprovante` e `listComprovantes` nao muda — callers continuam passando os mesmos parametros; a escolha de TTL e decisao interna de cada action.

6. `npm run lint && npm run typecheck && npm run build` passam sem erros.

7. Smoke test: gerar URL assinada via `listComprovantes`, aguardar 301 segundos e tentar acessar — deve retornar erro de URL expirada do Supabase Storage.

## Technical Notes

- API do Supabase Storage: `supabase.storage.from(bucket).createSignedUrl(path, expiresIn)` onde `expiresIn` e em segundos.
- Apenas a constante na linha 13 e as chamadas mapeadas acima sao afetadas. Nenhuma outra chamada de `createSignedUrl` foi identificada no codebase alem das 3 listadas.
- Proxy server-side (Next.js API route validando sessao + redirect para URL privada) e uma alternativa arquitetural mais robusta que TTL curto. Registrar como decisao futura no Change Log desta story e abrir story separada se o Dev avaliar necessario — esta story entrega apenas a mudanca de TTL.
- Complexidade: S / Risco S (mudanca pontual em duas constantes e uma chamada; sem alteracao de schema ou interface publica).

## Testing Requirements

- Smoke test manual (AC 7): validar expiracao real via Supabase Storage em ambiente de preview.
