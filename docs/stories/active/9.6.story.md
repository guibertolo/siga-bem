## Story
**As a** dono da frota,
**I want** acessar uma tela `/assistente` com chat onde posso perguntar em portugues simples sobre minha frota e receber respostas formatadas em streaming, alem de ver um item "Assistente" no menu principal do dashboard,
**so that** eu consiga tirar duvidas operacionais sem navegar por varias telas.

## Epic
Epic 9 — Assistente FrotaViva. Depende de 9.5.

## Acceptance Criteria

1. Rota `app/(dashboard)/assistente/page.tsx` criada como Server Component:
   - Verifica role via `getCurrentUsuario()`; se role = `'motorista'`, retorna 404 (nao autorizado) ou redireciona para `/dashboard`.
   - Renderiza componente client `<ChatUI />`.
2. Componente client `app/(dashboard)/assistente/chat-ui.tsx` criado:
   - Usa `useChat()` do AI SDK v6 apontando para `/api/assistente/chat`.
   - Exibe thread de mensagens (user + assistant) com streaming visivel.
   - Renderiza markdown + tabelas (usar biblioteca markdown ja presente no projeto; se nao houver, instalar `react-markdown` + `remark-gfm`).
3. Empty state: quando a thread esta vazia, exibe titulo "Assistente FrotaViva" + subtitulo explicativo em pt-BR + **as 5 perguntas-farol como botoes clicaveis** que, ao clicar, preenchem o input com a pergunta:
   - "Qual caminhao deu mais prejuizo esse mes?"
   - "Quais motoristas estao com CNH vencendo nos proximos 30 dias?"
   - "Me faz um resumo do desempenho da ultima semana"
   - "Quanto gastei de combustivel em marco?"
   - "Qual viagem teve maior margem?"
4. Input e botao de envio com **target minimo 48px de altura** (feedback_ux_older_audience). Texto minimo `text-base` (16px). Contraste >= 4.5:1.
5. Tratamento de erro: se o endpoint retornar 401 / 429 / 500 com JSON `{ error }`, exibir mensagem em um banner amigavel pt-BR **sem jargao tecnico**. Exemplo para 429: "Voce fez muitas perguntas em pouco tempo. Espere um minutinho e tente de novo."
6. Item de menu "Assistente" adicionado ao menu principal do dashboard (localizar arquivo correto via busca no projeto — provavelmente em `components/` ou `app/(dashboard)/layout.tsx`):
   - Visivel apenas para role `dono` e `admin` (nao para `motorista`).
   - Icone coerente com o design system existente (usar lucide-react ou o que o projeto ja usa).
   - Label: "Assistente".
   - Aponta para `/assistente`.
7. Loading state em `app/(dashboard)/assistente/loading.tsx` coerente com o resto do dashboard.
8. Error boundary em `app/(dashboard)/assistente/error.tsx` renderizando mensagem amigavel pt-BR.
9. Lighthouse audit manual em `/assistente` e em `/` nao regrediu: nas 4 categorias (Performance, Accessibility, Best Practices, SEO) o score permanece >= 95 em producao (ou >= 90 em dev — mas produtivo deve ser >= 95 conforme NFR-6).
10. Nenhum jargao tecnico visivel na UI ("Copilot", "LLM", "prompt", "token", "tool", "AI", "API" — tudo proibido).
11. `npm run lint && typecheck && test && build` passam.

### Markdown renderer
Verificar se o projeto ja tem `react-markdown` instalado. Se nao, `npm install react-markdown remark-gfm` (GFM habilita tabelas, que o spec requer). Adicionar ao package.json como prod dependency.

### Menu do dashboard
O @dev precisa localizar o menu atual — provavelmente `app/(dashboard)/layout.tsx` ou um componente `<Sidebar />`. Buscar por itens como "Viagens", "Caminhoes" para achar o padrao. Respeitar a gated-by-role existente no projeto.

### Bundle
A rota `/assistente` e code-split naturalmente pelo App Router. Nao importar AI SDK em nenhum componente global — apenas em `chat-ui.tsx` (client) e `route.ts` (server). Isso preserva Lighthouse na home (NFR-6).

### Teste manual obrigatorio
Fazer cada uma das 5 perguntas-farol via UI e validar visualmente:
- Streaming aparece token por token
- Resposta final tem tabelas/markdown quando apropriado
- Valores em R$ formatados corretamente
- Nenhum jargao tecnico aparece

### UX 55+
Lembrar: contraste alto, botoes grandes, texto claro, sem emoji decorativo (usa apenas se funcional), sem termo em ingles.
