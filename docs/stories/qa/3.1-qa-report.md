## Gate Decision

**Gate: CONCERNS**

Rationale: Story 3.1 viagem CRUD implementation is solid. Status transitions follow a well-defined finite state machine enforced at both the type and application level. Centavos handling is correct (INTEGER in DB, Math.round for motorista payment calculation). RLS policies correctly separate dono/admin from motorista access. The deferred FK pattern for gasto.viagem_id resolves the MIG-001 issue from Story 2.1. Viagem-veiculo sub-CRUD properly validates viagem editability before mutations.

The main concerns are: (1) no unit tests for critical business logic (status transitions, motorista payment calculation, veiculo validation), (2) 3 pre-existing TypeScript errors in the project that will fail CI, and (3) missing server-side validation that a caminhao is linked to a motorista via active vinculo. None of these are blocking for the current PR, but the TS errors (TS-001) should be fixed promptly as they affect the entire build.

**Recommendation:** Fix TS-001 (Zod enum syntax and FechamentoListItem mapping) before merge to unblock CI. Add tests for status transitions and viagem-calc as immediate follow-up.
